# Интеграция с Telegram-проектами в mm-системе (UI/UX и сессии)

Система `mm` поддерживает автоматическое обнаружение проектов на базе Telegram (как клиентских скриптов на `telethon`, так и чат-ботов на `aiogram`, `python-telegram-bot` и др.) и интегрирует в базу знаний правила для создания надежного кода и стандартизированного дизайна.

---

## 1. Безопасная работа с аккаунтами (Telethon)

Telegram жестко отслеживает параметры входа. Неправильный запуск купленного аккаунта (без прокси или с неверным фингерпринтом) приводит к моментальному бану. 

При обнаружении пакета `telethon` в зависимостях проекта:
1. В Obsidian Vault копируется подробная инструкция по сессиям [telethon-sessions.md](file:///C:/Users/louise/Desktop/louise-skills/templates/telethon-sessions.md) по пути `knowledge/integrations/telethon-sessions.md`.
2. В паспорт проекта (`passport.md`) в **Раздел 8. Контекст для промптов ⚠️** автоматически добавляются правила:
   * *При написании кода для Telethon-аккаунтов строго следуй правилам из [[telethon-sessions|инструкции по сессиям Telethon]].*
   * *Запрещены параллельные запуски сессий с одним файлом .session (риск AuthKeyDuplicatedError).*
   * *Все параметры устройства (fingerprint) бери строго из соответствующего .json файла.*
   * *Используй SOCKS5-прокси с rdns=True для всех подключений.*
3. В `project-instructions.md` для ИИ добавляется требование руководствоваться данной инструкцией.

### Отключение проверки UI/UX (Собственный дизайн)

Если вы хотите спроектировать интерфейс бота по собственным правилам и не хотите, чтобы `mm-doctor` выводил предупреждения о несоответствии дизайн-системе:
1. Откройте файл `passport.md` в корне вашего проекта.
2. В блоке frontmatter (в самом начале файла между `---`) установите параметр:
   ```yaml
   telegram_ui_ux: false
   ```
3. При следующей диагностике (`/mm check` / `mm-doctor`) все UI/UX проверки ботов (Phase 6.3) будут автоматически пропущены.

## 3. Диагностика и автоисправление (`/mm check` / `mm-doctor`)

Команда диагностики системы [mm-doctor](file:///C:/Users/louise/Desktop/louise-skills/skills/mm-doctor/SKILL.md) (запускаемая через `/mm check` или `/mm-doctor`) автоматически проводит статический анализ кода Telegram-проектов на наличие критических уязвимостей и рисков блокировки.

### Выполняемые проверки:
1. **Для Telethon**:
   * Наличие файла `knowledge/integrations/telethon-sessions.md`.
   * Отсутствие прокси в вызовах `TelegramClient(...)`.
   * Жестко захардкоженные параметры устройства (fingerprint).
   * Отсутствие проверок аккаунта на freeze после входа.
   * Отсутствие перехвата ошибок `FloodWaitError` и `AuthKeyDuplicatedError`.
2. **Для чат-ботов (UI/UX)**:
   * Наличие файла `knowledge/patterns/tg-bot-ui-ux.md`.
   * Создание `ReplyKeyboardMarkup` без параметра `resize_keyboard=True` (что раздувает клавиатуру).
   * Использование небезопасного `MarkdownV2` для вывода динамических данных (рекомендуется HTML + `html.escape()`).
   * Наличие жестко захардкоженных ID, токенов или ссылок в текстах без оборачивания в моноширинный тег `<code>` (для копирования в 1 клик).
   * Неправильный порядок кнопок в inline-диалогах подтверждения (норма: `[✅ Да] [❌ Отмена]`).

### Безопасное автоисправление (Auto-fix):
Прямое изменение кода клавиатур и FSM ИИ-агентом без контроля разработчика опасно из-за риска сломать роутинг хендлеров. Поэтому `mm-doctor` предлагает безопасный сценарий:
1. Генерирует в проект хелпер-модуль `tg_bot_ui_helper.py` с готовыми безопасными конструкторами и middleware.
2. Выдает разработчику пошаговые diff-инструкции для ручного импорта и внедрения.
